1019 - iPic 换『芯』上架

发表于 2016-10-19 | 分类于 每天写一点

如果你 在 macOS App Store 下载 iPic，可能会觉得和之前 iPic 并无太大不同。但其实，iPic 内在已经有了很大变化。

iPic + Swift 3.0

是的，第 1 个主要的变化就是，iPic 已经适配了 Swift 3.0. 说实话，这对于普通用户确实是透明、无感的。但了解 Swift 3.0 的朋友会知道，这样的适配代码会有大量的调整，还有很多隐藏的坑。并且，由于代码的变化，App 每个犄角旮旯的功能都需要进行仔细的测试。

iPicUploader Inside

第 2 个主要的变化是，集成了 iPicUploader. 同样，对于 iPic 的普通用户来说，这是透明、无感的。不过，集成 iPicUploader 的意义在于：别的 App 也可以通过调用 iPic 来上传图片、并保存 Markdown 链接。

比如，你偏爱某款 Markdown 编辑器、而它又不支持上传图片。你可以邀请编辑器的作者来集成 iPicUploader. 这样，你在使用 Markdown 编辑器写作时，只要将图片拖拽至编辑器，编辑器自动调用 iPic 上传图片、保存 Markdown 格式链接、插入图片，一切就像将图片拖入 Word / Pages 中自然。

目前，已经在和某款 Markdown 编辑器（暂不提名字）合作，等这个产品新版上线了，再来告诉大家。如果你有想支持的 App，也欢迎和我、或者和 App 的作者联系。

功能上的变化

支持七牛华南、北美节点

同样，对大家是透明、无感的，只要在配置七牛图床时，填写 Bucket 即可。对于如何区分 Bucket 所在地区，交给 iPic 来做好了。

支持复制图床

为了安全，图床的 Secret Key 信息是使用 * 隐藏显示的。而对于 S3 这种一旦生成 Secret Key、就不能再次查看的服务，就会有些不便：除非在某个地方记录 Secret Key，不然就无法使用其添加多个图片。

现在，有了图床的「复制」功能，只要复制已有的图床、然后再编辑 Bucket 名称即可。安全与便利兼得。

马上升级

下载图床神器 iPic

如果你觉得 iPic 还不错，麻烦下载后顺手 给个好评 之前好评过的朋友，也麻烦更新下评论内容（这样评论才会出现在最新版中）你知道这对 iPic 很有帮助，也能让更多的朋友发现 iPic；谢谢你的帮忙。

当然，如果你想要反馈或者吐槽，都欢迎 跟我聊聊

希望 iPic 能成为你使用 Markdown 写作时的插图好帮手。

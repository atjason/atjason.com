0309 - 持续学习

发表于 2023-03-09 | 分类于 每天写一点

单一技能，想吃一辈子，不可能。

一方面，活到老学到老。

另一方面，会的东西越多，能组合的也越多，可能性越多。

比如，前后端通吃，软硬件通吃，设计与实现通吃。

不想学可以不学，但不能拦着别人学，机会被别人抓住了别眼红，过得不如别人也别怪社会，那也可以。

当然，这些是有大前提的，社会是整体公平的。别跟生下来，起跑线就在你的终点线那种人比。

现在应该没人问学英语有没有用、学计算机有没有用了吧。

你在写稿掉头发时，人家 ChatGPT 早把活干完了，你说学不学。这时候，学习不代表进步，而是追赶、别被甩太远。

很多 Python 课程都在贩卖焦虑，什么你加班别人早用 Python 干完活下班。可那种级别的焦虑，跟 ChatGPT 这种碾压，不是一个级别。

换个角度，学会使用 AI 这种工具去实现，而不是自己花时间实现，做善用工具的人。

0308 - 贫穷容易黑化

发表于 2023-03-08 | 分类于 每天写一点

里约黑帮大佬，起初也是老实人。黑化的过程和高启强很像：生活所迫。

为什么政府要关注民生、就业，因为如果一堆人为生活所迫而黑化，那整个不乱套了。

打败高的是安欣，

消灭高的是人民币。

0307 - 领导与管理

发表于 2023-03-07 | 分类于 每天写一点

管理是正确地做事，
领导则是做正确的事。

管理是有效地顺着成功的梯子往上爬，
领导则判断这个梯子是否搭在了正确的墙上。

管理者关注的是实现、问题，
领导者关注的是方向、机会。

0305 - 千头万绪，抓住重点

发表于 2023-03-05 | 分类于 每天写一点

智能跑轮的项目，到了推进生产、启动运营推广的阶段，各种采购备料、电路板和外壳配合、包装与说明书、文案及配图、宣传视频、确定颜色与备货、等等，一大堆的事，想想都头疼。

其实，也是第一次应对这种情况。不过，事情也都是相同的。

一方面，抓住重点。比如，现阶段生产就是最重要的。确定大概可行的内测公测时间，然后全力去推，遇到问题解决问题。

另一方面，不同的事，重要性是不同的，所花费的时间精力也是不同的。极端点说，有的点不做其实也没关系，而有的点不做则会卡其他所有事。

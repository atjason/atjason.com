0602 - 共享工具？

发表于 2017-06-02 | 分类于 每天写一点

最近共享概念还挺热，比如共享雨伞、共享篮球、等等。像这样的东西都可以共享了，还有什么不能共享？

其实，我倒是挺希望有 共享工具 的。比如，前段时间家里需要钻个孔，我到楼下做铝合金窗户的店铺里借电钻，结果去了 3 次，人家以各种理由推脱，就是没借到。最后，我自己在淘宝买了一个。用完之后，就一直闲置在家里。

如果当时有共享这个电动工具的，我就是挺愿意一用。而目前我有了之后，我也愿意租给别人用，但并不愿意为此花时间、精力。共享电动工具这事，问题还挺多，比如如何取、送、保护隐私、等等。

0601 - 管理期望值

发表于 2017-06-01 | 分类于 每天写一点

东西贵不贵、好不好，是相对的；相对于什么呢？相对于用户的期望值、心理预期。

要「管理」用户的期望。

比如，软件的免费 + 内购模式。起初，用户看到内购价格时，可能会觉得贵。但看的多了，慢慢就不觉得贵了。某一天需要的时候，可能就入手了。

比如，坚果 Pro 发布会上将产品吹的天上手、地下无的，可当拿到手，看到背板不平这样低级的质量问题，还是会有种吃苍蝇的感觉；心理落差大。

好坏无绝对，只能相对满意。

0531 - 两眼论

发表于 2017-05-31 | 分类于 每天写一点

围棋中有「两眼即活」理论，可以简单理解为：多一条路，就不会死。

对应到自己的生活中，发现可以找到很多映射：

领域 A B
操作系统 macOS Windows
编程语言 Swift Python
Python 虚拟环境 pyenv virtualenv
手机 iPhone 锤子
服务器 阿里云 Vultr
CDN 七牛 CloudFlare
通讯 微信 Telegram
… … …

换个角度讲，不在一颗树上吊死。多了解一个领域、多一个选择、多一种跨界，就多一种活下去的方式，就更不容易死掉。

0530 - Radxa 又复活了

发表于 2017-05-30 | 分类于 每天写一点

最近，项目上需要一个能 24 小时运行的桌面版操作系统，像 Raspberry Pi、Radxa 这样的板子，简单是太适合了。于是，拿出吃灰已久的 Radxa，重新刷新了 Ubuntu Desktop，配置了一下环境，完美运行所需要的小程序，爽！

平时积累的素材、能力，总会有需要的时候。当然，如果能和大方向关联，就更实在了。比如，Facebook 创始人扎克伯格每年的习惯：学中文、每天写代码、每天打领带、每天认识一个公司以外的人…都是和公司战略想契合的：连接地球上每一个人。

提前布局，坚持在各个点上做准备，终会连点成线、结线成面，形成战略优势。

0529 - 这几天状态不错，开始有对的感觉

发表于 2017-05-29 | 分类于 每天写一点

首先，确认了坚持做独立开发者，至少到年底。

进而，明确本年度的技能树培养：macOS/iOS/后端

进而，技术上开始布局：

  • 后端方面
    • 学习 Python
    • 搭建国内外服务器，继续学习 Linux，学习 CDN 等网络工具
  • iOS 方面
    • 使用坚果 Pro 作为主力机，了解 Smartisan OS 的特性
    • 下半年使用 iPhone 8 作为主力机

有了前面的布局，产品方面可以进行拓展：

  • Klib
    • 可以考虑更方便地共享笔记
  • iPic
    • 考虑 iOS 端
  • 全新产品
    • 支持 macOS、iOS，有后端做保障，有更多可能性

恩，开始有做对事的感觉了。

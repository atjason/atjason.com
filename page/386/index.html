0701 - 不作死就不会死

发表于 2016-07-01 | 分类于 每天写一点

今天干了件很晕的事：不小心发布了还没做好的 iPic. 当时那个郁闷啊，科想把手续剁了。

终于，还是找到了补救的办法，不至于让事情太糟。

其实，也没什么大不了的，天不会塌，即使出了问题也总有补救的办法。

太累了，就不要干活了，至少不要干关键的、重要的活。不然，出了问题，弥补的时间会远大于做这点事所带来的收益。

0630 - 优雅的代码

发表于 2016-06-30 | 分类于 每天写一点

今天整整写了一天代码。虽然还想写，无奈眼睛和脖子受不了了。可惜的是，Imgur 还是没有全部搞完，虽然主体已经好了。

早上的时候想的挺好，计划上午搞完，后来推迟到下午，结果晚上也没搞定。又一次认识到：从通过技术可行性研究，到写出优雅的代码、尤其是在复杂业务情况下的优雅代码，差得远。

回顾今天做的事，整体的节奏、方法还是对的，虽然也多走了一些弯路。可能还是经验和能力的问题的，感觉只需要一半的时间就应该能完成今天干的事。

继续磨练，坚持、继续加油！

0629 - 拳不离手

发表于 2016-06-29 | 分类于 每天写一点

这几天一直忙测试、写文案、推广，好几天都没写代码了。

这个状态是不对的。毕竟我本质上是开发者，最核心的能力还是写代码。首先要把东西做出来，这都是一切的根本，是 1000… 前面的 1 。有了这个 1，后面的运营、推广这些 0 才能意义。

另外，总感觉 Imgur 的实现是半成品，都不好意思去推广。明天，充分实现 imgur 的集成，后天晚上重新提交。

0628 - 最难的时刻：推广

发表于 2016-06-28 | 分类于 每天写一点

iPic 昨天提交上架 Mac App Store、今天被拒。不过不是很担心，已经改好又提交了，应该问题不大。

问题大的是，如何推广。毕竟开发都在自己的设计和掌握中，出不了太大的问题。而运营、推广，则完全在自己控制范围外了。虽然国内有少数败家媒体可以帮忙宣传，但在国外就几乎是一片空白。只能像无头苍蝇一样到处去找 Mac 论坛、网站，然后发帖、发邮件，完全不知道对方会不会回复。

看来自己之前在运营上下的功夫太少了，现在到了补课的时候了。今天就这样吧，期待明天醒来看到审核通过的好消息，然后继续研究如何做海外推广。

0627 - 提交上架总是有些慌乱

发表于 2016-06-27 | 分类于 每天写一点

今天提交 iPic 上架 Mac App Store.

虽然之前已经上架了 3 款 Mac App，但在上架前后，问题有些慌乱。毕竟事情太多太杂，比如 App 名称、关键字、描述、截图、测试、运营文案、等等。相比而言，在开发的时候，虽然难度大些，但一次只需要处理一件事，更专注、也不慌乱。

也许这也是难以避免的，甜蜜的麻烦吧，毕竟看着自己辛苦做的 App 终于要面世了，心里还是很开心的 ^_^

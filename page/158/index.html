0114 - 注意力管理

发表于 2017-01-14 | 分类于 每天写一点

为什么我们总是喜欢把时间花在无聊的事上？

比如我，今天的大部分时间都花在了 iKindle 开发上；间隙时间（比如在马桶上），并不想去看一些费脑的东西（如长篇文章、英文、需要记忆或操作的内容、等等），而是更愿意看一些轻松的（如新闻、搞笑视频、朋友圈、等等）

当然，我也知道做这些事情，更像是在打发时间，对要做的事、对自己的提高没什么太大帮助。可为什么管不住自己呢？

问题出在注意力上。

我们的注意力是有限的，或者说，在有限时间内是会消耗殆尽的。当缺少注意力时，我们就不愿意再做些费脑的事了。

那怎么办呢？就是要管理注意力。

比如，注意力有个特性：可以较快恢复。对此的应用是，把大块的任务尽量分成适当的小块，每做完小块任务后，放松一下；连续做了几块任务后，彻底放松下。有没有听起来很熟悉？没错，这就是番茄工作法的原理。

另外一个就是：在注意力集中时，做需要注意力集中的、比较复杂的事。要做到这一点，又和任务管理联系起来了。因为，只有任务管理的好，才可能在注意力不集中时，很快地找到不太需要集中注意力就能做的事。这是目前自己做的还不好的地方。可以利用 OmniFocus 的场景功能来标注此类任务。

0113 - 保持创意

发表于 2017-01-13 | 分类于 每天写一点

除了 iKindle 开发，今天最大的收获来自这篇 对「极光词典」作者的采访

保持创意

我的 Reminders 中有一个「创意」列表。每当我有一个新需求，或想到一个新创意，我就会在这个列表中创建一条新 todo 并记录下来。

记录下来后，或有空闲时间时，我会通过 Google 和 App Store 寻找现成的解决方案。如果有并且能做的很好，我可能就会使用那款产品，并把创意 todo 标记为完成。如果没有，我则会考虑自己来实现。

恩，这个方法好，立马在自己的 OmniFocus 中加个「创意」清单。

当创意不那么新

另一个重要的收获来自微博上这篇报道的评论：

所以这个事情只说明一个道理：不要因为市场上已经存在了什么，就不去做了~也不要总想着做点别人没有想到的~

确实是这样。比如，在作者开始做词典时，市场上已经有大量的词典 App，还要做吗？作者如是说：

我理想中的词典应用的使用流程很简单：1. 输入词句；2. 查看释义；3. 查下一个（如果有）。但是，市面上的词典应用被我用了个遍后，我就发现，能把这个流程做到流畅的几乎没有，仅有的一两个又没有足够美观的界面。于是我就为我自己打造了极光词典。

所以，创意新不新不是最重要的；重要是，自己对产品有没有更深的理解，是否能通过自己的打磨把产品做到极致。而要达到这一点，作者的一个理解是：

做你真正想做的产品，做你自己也需要使用、也乐于使用的产品。

总体看下来，发现和作者有很多相同的理念，决定勾搭一下。

0112 - 少做非标件

发表于 2017-01-12 | 分类于 每天写一点

在做 iKindle 项目时，有个点是预览书籍。本来最佳实践是使用系统的 Quick Look，无奈这路子还挺复杂；并且，可能也会在沙盒的时候遇到麻烦。

于是乎，开始自己模拟实现 Quick Look；
于是乎，开始入坑了…

界面样式、交互、动画、等等，看起来很简单的东西，但真要做出来，还是很难的。关键，这部分只是产品的一小部分，为了这一小部分花费大量时间，并不划算。并且，由于很难 100% 模拟，做出来的交互会让人感觉很廉价、不专业。

现在大的样子已经做出来，暂时先不动了（既不切换至 Quick Look，也不继续增加相似度），等体验版做出来的，到时看大伙的反应再说吧。

恩，尽量少做非标准件。

0111 - 数组与链表

发表于 2017-01-11 | 分类于 每天写一点

不是在说数据结构，而是在说两种模式、状态。

数组模式，在一开始就知道所有：类型、地址、容量、等等。优点是效率高，缺点是不易变动。

链表呢，很不同。在意的是每时每刻，不知道未来的路有多长，也不一定知道过去；而只在意当下、与下一步。有此两样，但可开路搭桥、一往无前。

想想，我更多的是链表模式。

原来，人生也是由数据结构和算法组成的呀 😂

0110 - 一次只做一件事，做好

发表于 2017-01-10 | 分类于 每天写一点

本来这两天在很开心地开发 iKindle，结果半路杀出来 iPic…

这两天爆出来，iPic 在上传至默认的微博图床时，有可能失败。在几位热心的 iPic 用户帮忙之后，终于解决了这个问题。

可是，有个比较纠结的事：既然已经出 iPic 新版了，要不要顺便再做点功能呢？如果做，就需要花时间，并且会更长时间地打断 iKindle 的进展。如果不做，有可能使 iPic 的迭代周期过于频繁，对于程序的稳定、及运营并不利。

最后决定：尽快上架 iPic，解决用户的问题，这是第一位的。然后抽身，全力继续 iKindle 开发。目前 iPic 新版已提交审核，期望明天能顺利上架。

1228 - 控制自己的情绪

发表于 2018-12-28 | 分类于 每天写一点

良性循环与恶性循环，可能会互相转换的。情绪，可能是转换的一个标志点。

做事有成绩，心情好、效率高，然后继续保持高效、继续有成绩。

而一旦做事遇阻、不明朗，人就会消极，就会觉得诸事不顺，开始发脾气，效率也下降，做很多无用功，然后做事继续受阻。

一旦觉得情绪变糟，就要警醒。要担心的不是做事受阻，而是坠入恶性循环。

与通胀相比，滞胀更可怕。

1227 - 事前设好期望结果

发表于 2018-12-27 | 分类于 每天写一点

跟目标很像，但味道又不一样。

产品、事情能做好到什么程度，是永无止境的。而我们的时间精力能力又都是有限的。如果事先不大致想好这事做到什么程度我就满意，那就永远无法满意。

没有目标就没有效率，没有期望就不会满意。

1226 - 不过份自责

发表于 2018-12-26 | 分类于 每天写一点

如果一件事没有好的结果，一定是有自己的因素。但，也一定不全是自己的因素。

想清楚自己做的不好的地方、下次如何改进，然后原谅自己。

注意：原谅自己很重要。不然，自己会一直陷入自责，让过去的错误持续伤害现在的自己。

1225 - 选择环境

发表于 2018-12-25 | 分类于 每天写一点

和原来生活的城市相比，最近一直在一个相对内陆、更多线的城市生活，能明显感到很多差异；比如，快递不能隔天到。

环境，是远大于个人能力所能控制的。

  • 比如，你可以不随地吐痰，但你管不住路人在你不远处吐痰。
  • 比如，早餐你想吃得健康，但即使有钱你也无法忍受每天跑很远的地方去买所谓的健康早餐。
  • 比如，医疗、教育、等等

选择好的环境，能拔高起点，节约很多隐性成本。

1224 - 吐槽支付宝小程序

发表于 2018-12-24 | 分类于 每天写一点

最近在搞支付宝小程序，有很多不爽的地方，吐槽一下：

  • 支付宝的会话过期时间真短。扫码登录，上个厕所回来，对不起，登出了，又得扫码。
    • 这也是定位含糊引起的。支付宝本身是个支付工具，安全性要求高，会话很快过期没问题。可它又同时承载了小程序及其开发工具，就很难受了。
  • 授权机制不合理。如果用户拒绝了授权，并不能通过某个机制查到这个状态；且，还是「有可能」再次弹出授权对话框。这个「有可能」，则由所谓的疲劳机制确定的。这种不确定的机制，给逻辑带来挑战。
  • IDE
    • 预览小程序时，需要将小程序上传至阿里的某服务器。这一过程，很慢。甚至能达到 30s 之久，无法忍受。对比隔壁家的微信小程序，可以使用快捷键推送至手机、无需扫码，方便不要太多。
    • 只有黑色主题，不喜欢也没办法，白天看不清楚。调试器又是白色的；为了看清楚黑色部分，把亮度提高，打开调试器，瞬间亮瞎眼。
    • 快捷键不完善，比如没有呼出提交 Git 代码界面、输入 comment 后直接提交的快捷键。
    • 调试器不能查看 PageData
    • 不能将预览界面放在左侧，和微信 IDE 相反，难受。好吧，也怪微信不能放右边。
  • 只能添加 5 个域名，不太够。
  • 不对个人开放支付宝小程序。
  • 审核机制很奇葩。
    • 我的小程序名字是「实时公交查询」，被拒，说是名字太宽泛，我去。一个叫「实时公交」的小程序都已经上架了。
    • 我选的分类是「公共交通」，说不合适，要选「生活服务」，我去，公交查询不算公共交通分类的？！
  • 想和用户建立联系，只有用所谓「智能客服」，收费不说，开通过程各种出错。且极复杂，是企业级的，跟小程序不搭。
    • 隔壁家的微信，免费快速，还能在微信里直接回复，体验佳。
  • 性能明显不能微信。相同的手机、相同的代码，支付宝明显界面卡顿、刷新慢。
  • 删除小程序，小程序的数据不会删除；至少在可见的一段时间，并没有删除。
  • 跟微信小程序很像，又有很多地方不一样。对于想要同时开发微信和支付宝小程序的人，很折腾。

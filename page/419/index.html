0208 - 我还是不太理解开源世界

发表于 2018-02-08 | 分类于 每天写一点

最近在了解币圈的东西，了解到 ccxt 这个库，看起来很不错。

从 README 来看，这个库在不同的币圈交易所之上，抽象了一层通用接口，不仅封装了各个交易所不同的 API 接口，还统一出了一致的 API，目测用起来会很简单。不仅如此，其文档也很齐全。我看了一部分，扫清了自己的一些知识盲点。

这样一个优秀的库，是如何生长起来的？数据上看，目前有 1 万多次提交、127 位参与者、首个版本发布于 2017 年 8 月 1 日。是什么吸引了这么多优秀的人，花这么的时间，免费打造这样一个优秀的库？对于我这俗人来看，确实很难理解。

如果我确实会用到它、且它对我有用，我会捐赠的，立 flag; 以及类似的库我也会这样。虽不理解，但我依然很希望这样更美好的事，越多越持久越好。

0207 - 靠什么赚钱？

发表于 2018-02-07 | 分类于 每天写一点

同样是赚钱，依靠的东西并不一样。

  • 靠体力
    • 比如工地工人；一些技术含量低的，也可以说是靠体力。
  • 靠技术
    • 比如程序员，本质上是靠技术，出卖自己的时间赚钱。
  • 靠资本
    • 比如投资家，基本是钱生钱的生意。
  • 靠品牌
    • 比如公司产品，即便不如竞品，但只要有品牌、存量用户，依然能活得不错。

你，靠什么？

0206 - 在没时间时准备，在有时间时干

发表于 2018-02-06 | 分类于 每天写一点

最近在家带娃，能干活的时间不多。而如果等有时间了再考虑做什么、怎么做，就太慢了，毕竟时间太少了。

应该在带娃的时候，就要在脑子里想问题。比如，接下来要做什么，准备怎么做，可能会遇到什么问题，等等。有了这些铺垫，等有时间时，就可以直接上手干活，不至于在想的过程中浪费时间。

0205 - Klib 救了一天火

发表于 2018-02-05 | 分类于 每天写一点

Klib 从 Amazon 导入标注，逆向了官方的一些接口。可最近，官方的接口有变化，导致 Klib 导入失败。

今天紧急救火，终于还是解决了。其实还是挺麻烦的：

  • 逆向本身有些麻烦，要用抓包工具、看 html/js 等源文件等方式，甚至还要加上猜测与运气的成分，才能跑通。
  • 更关键的，这样只能覆盖自己本身的情况。而用户的数据不同，程序流程上可能会进入不同的分支；很可能逆向时并未覆盖这一分支，就会导致失败。

虽说目前算是解决问题了，但明显这样的感觉很不好：要是大年三十出问题了，我总不能不吃年夜饭吧。现在心里大体有更靠谱的方案了，不过需要大块时间来实现。考虑到马上要过年，就很不折腾了，下次出问题时再彻底替换吧。

0204 - 看别人的代码，真累

发表于 2018-02-04 | 分类于 每天写一点

自己的写的代码，不管好坏，都是自己写的，毕竟熟悉。即使是对于复杂的逻辑，通常是在简单的逻辑上慢慢生长的，所以也算容易理解。

而直接看别人的代码，尤其是复杂的、本身结构不太优的、夹杂着一些自己的不懂的技术的，看起来真是累啊。

现在只能这样尝试：

  • 硬着头皮看。看到不会的东西，做个标注。如果已经做的标注实在太多、看不下去了，就回过头来一个个搞定这些标注。
  • 对于相同的功能，尝试自己设计，看自己会怎么做。然后对比别人的做法，更容易理解其这样做的目的，发现其中好的、不好的地方。

但依然很累，催眠效果很棒…

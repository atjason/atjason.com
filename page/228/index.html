0224 - 微信读书中的「热门图书 + 热门标注」

发表于 2021-02-24 | 分类于 每天写一点

前几天在 Klib 中增加了导出微信读书笔记的功能：链接

做的过程中，就想整这么个东西：把微信读书中热门图书的热门标注，集中整理出来，一次看个够。

热门标注还是挺好玩的；像是众多读者一起，把书中精华的部分拎出来，三分钟看完一本书。在决定是否精读一本书时，除了看评分，还可以看看这些热门标注，看是否对自己的胃口。

比如，我看到 这段话，就笑了：

“轻车熟路，自己慢慢生吧，”上官吕氏把一卷白布、一把剪刀放在炕上，蹙着眉头，不耐烦地说，“你公公和来弟她爹在西厢房里给黑驴接生，它是初生头养，我得去照应着。”

欢迎一试：http://klib.me/weread/hot/

0223 - 什么是吃苦

发表于 2021-02-23 | 分类于 每天写一点

大多数人对吃苦的含义，理解得太肤浅了。

穷，根本不是吃苦。
吃苦，不是受穷的能力。

吃苦的本质，是长时间为了一件事聚焦的能力；以及，为了长时间聚焦的做一件事的过程中，所放弃的娱乐生活、所放弃的无效社交、所放弃的的无意义的消费生活，以及在这个过程中所忍受的孤独和不被理解。

吃苦，本质是一种自控能力、自制能力和坚持能力，以及深度思考。

很大程度上，靠自己成功的富人，往往比穷人能吃苦；否则，他就不能靠自己白手起家。你会发现，他富了以后，做事情还是比你勤奋、还是比你能忍受孤独、还是比你更有理想。

这才是吃苦。

Klib 是什么？

0220 - 推广比写代码还累

发表于 2021-02-20 | 分类于 每天写一点

写代码实现功能，基本是封闭问题。就是说，自己知道目标是什么、达到目标前有哪些问题或困难，然后一一解决便是。

而运营推广，是开放问题。目标不确定；达到可能的目标，又有无数种途径。对于开放性问题，以及流量运营，好像我不太擅长。需要做的选择取舍太多，多渠道追踪起来又很花时间。

并且，准备运营的素材，包括文案、图片、视频或动图等，也是非常花时间、花精力的。比如，如果图片中需要出现产品的界面，对于内容型产品，就要事先准备大量优秀的内容，以填充应用界面。最后，看起来是一张简单的图，背后却有大量的工作。如果是视频，要求则又高一个量级。

之前，我一直想，要是有人能帮我把运营推广给做起来，我专注于产品本身多好啊。一方面，这样的合作伙伴并不好找。另一方面，运营的过程，也是和用户深入交流的过程。这样关键的环节，也不能完全假他人之手，还是要自己花时间参与的。
